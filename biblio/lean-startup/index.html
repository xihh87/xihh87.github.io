At the root of every seemingly technical problem is a human problem. (relacionado: cada problema de negocio es un problema personal disfrazado [Toilet Paper Entrepreneur].)

  • Incluir en la discusión a todos los involucrados, incluyendo quién descubrió, diagnóstico, atendió, servicio a cliente, todos los que trabajaron en los sub-sistemas o características.
  • Si el problema escaló, los tomadores de decisiones deberían estar presentes. (si alguien no está en la habitación suele ser el chivo expiatorio.)

If a mistake happens, shame on us for making it so easy to make that mistake.

  • Se tolerante de cualquier error la primera vez.
  • No permitas que el mismo error ocurra una segunda vez.

Este sistema de dos reglas tiene problemas:

  • Es ambiguo si se debe prevenir una instancia del problema o toda la categoría.
  • La gente suele quejarse de que no hay tiempo para arreglar de raíz los problemas.
  • La gente va a llevar el proceso a echar culpas.

Alguien con autoridad debe cuidar que este proceso no se vuelva problemático. (Empezar por el nivel de abstracción más bajo.)
