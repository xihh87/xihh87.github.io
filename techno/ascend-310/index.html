Cómo aprovechar los NPU del Atlas 800

Resulta que administro un Servidor de Inferencia Atlas 800 que tiene un Neural Processing Unit Atlas 300I.

El NPU supuestamente tiene:

  • capacidades de codificación de video
  • superpoderes para calcular redes neuronales

Implementé una prueba de concepto para mostrar la inferencia usando OpenCV para detectar imagenes. Funciona, pero no utiliza la NPU.

  • ☐ Sospecho que se tiene que instalar una versión específica de OpenCV2.

    Las instrucciones para Atlas 200 DK (traducción)piden instalar localmente:

    • ffmpeg-4.1.3
    • OpenCV
    • OpenCV Contrib
    • compilar python-cv2
    • configurar LD_LIBRARY_PATH para cargar primero las bibliotecas personalizadas

    Las instrucciones para Atlas 300:

Prueba de concepto en proyecto de Inferencia ↑

  • Generar una imagen para entrenamiento que incluye las dependencias para desarrollo.

  • Entrenar modelo

Parece que existe un adaptador que se puede usar en OpenCV2 que proviene de pyACL (ascend computing language), que no está disponible en PyPI.

pyACL se instala como parte del entorno de CANN siguiendo alguna de estas guías, o éstas:
