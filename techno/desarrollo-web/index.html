Mejorar la seguridad de un sitio web ↑

Tune the web, explica cómo implementar Content Security Policy (CSP).

Me parecieron importantes estos ejemplos que propone.

Para detener los ataques XSS:

Header Always Set Content-Security-Policy \
    "default-src 'self' https:; \
    script-src: 'self'; \
    style-src: 'self';"

Para reportar todos los usos de código extraño en tu web:

Header Always Set Content-Security-Policy-Report-Only \
    "default-src 'self' https:; \
    script-src: 'self'; \
    style-src: 'self'; \
    report-uri: https://fqdn/path/to/report"

La política completa que usa en su sitio:

Header always set Content-Security-Policy-Report-Only
    "default-src 'none'; \
    base-uri 'self'; \
    font-src 'self' https://a.disquscdn.com; \
    script-src 'self' \
        https://a.disquscdn.com \
        https://cdn.ampproject.org \
        https://cse.google.com \
        https://translate.googleapis.com \
        https://translate.google.com \
        https://tunetheweb.disqus.com \
        https://www.google-analytics.com \
        https://www.googleapis.com \
        https://www.google.com \
        https://www.gstatic.com;
    style-src 'unsafe-inline' 'self' https:;
    frame-ancestors 'none'; \
    form-action 'none'; \
    img-src 'self' https: data:; \
    connect-src 'self' \
        https://www.google-analytics.com; \
    child-src 'self' https://disqus.com; \
    frame-src 'self' https://disqus.com; \
    media-src 'none'; \
