En 2004, un escritor que estudió ciencias computacionales, llamado John Gruber, desarrolló un lenguaje para escribir fácilmente páginas web con sus elementos más comunes.

Tiene pocos elementos, es fácil de usar, es legible directamente, entre otras ventajas.

Antes de markdown hubo muchos, pero markdown es especial porque está diseñado para ser fácil de leer y escribir, y se puede publicar tal cual.

Esa simplicidad ha hecho muy popular el formato, y la gente que empezó a usarlo notó que tenía gran potencial y pequeños problemas. Cuando lo comentaron en la lista de correos, la respuesta de Gruber fue «markdown es un producto terminado».

Entonces varias personas hicieron lo que cualquier hacker que se respete: resolver sus problemas por sí mismos y compartir la solución.

El problema es que varias personas resolvieron esas cuestiones en formas diferentes, generando varios dialectos de markdown…

La mayoría de los sistemas son compatibles con CommonMark ↑

CommonMark es una especificación completa del formato, que a diferencia del markdown original evita ambigüedades y tiene una única descripción en cada caso.

Está basado en un consenso entre los diferentes motores de markdown.

Define un mínimo de compatibilidad y puede ser un buen piso para generar extensiones.

Si requieres que tu markdown funcione en cualquier lado (o varios) tu mejor apuesta es limitarte a CommonMark

Sin embargo, a mi entender el dialecto más interesante de markdown es

Pandoc ↑

John McFarlane programó pandoc para transformar archivos entre muchos formatos y es uno de los pocos programas que me encantan.

Markdown en telegram ↑

Telegram usa una versión modificada de markdown que permite algunas extensiones. No es compatible con CommonMark.
