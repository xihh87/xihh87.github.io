Me gusta escribir usando mi editor de texto, markdown y pandoc.

La principal ventaja es que puedo convertir ese texto a muchos formatos y gran parte de internet acepta este formato en sus páginas web.

Uso esto a veces para hacer presentaciones y tal vez me gustaría hacer automáticamente el resumen que la acompaña.

(Para el formato uso principalmente latex, que no me gusta pero esa es otra historia.)

Últimamente uso metodología DevOps para mis proyectos incluyendo todo lo que escribo.

Por eso estuve buscando herramientas para verificar gramática automáticamente.

Herramientas libres ↑

  • El manual de cocina de linux menciona herramientas como style y diction.

  • textidote revisa LaTeX directamente

  • after the deadline usa aprendizaje de máquina para revisar la gramática, acepta varios idiomas e interactúa con varias herramientas (como languagetool y ).

  • proselint

  • vale syntax aware linter for prose recomendado por semipol.de.

  • check-style que revisa las reglas preferidas de un programador

  • gramma envía a un servicio externo tu texto para revisar y tiene una GUI para corregir los errores.

Herramientas no libres ↑

  • Grammar Bot es un api para revisar gramática y ortografía.

Verificar LaTeX ↑

  • lacheck encuentra errores comunes en LaTeX.
