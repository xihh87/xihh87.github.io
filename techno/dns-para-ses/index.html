Cómo registrar los nombres de dominio para enviar correos con SES

  • Cómo funcionan los dominios para intercambio de correo (MX)

La documentación oficial dice que debes agregar los dominios desde la consola de SES.

Hay que agregar 4 entradas al DNS para acreditar que posees el dominio.

  • Un TXT _amazonses. con una cadena aleatoria

  • 3 CNAMEs para habilitar el formato de correo.

    Al editar estos dominios, es importante poner un punto al final para que los dominios entren correctamente.

Se puede usar esta herramienta en línea para verificar la propagación de los registros

Al parecer no pueden usarse dominios de correo diferentes.

Cómo funcionan los dominios para intercambio de correo (MX) ↑

Los dominios de intercambio de correo (MX) tienen el formato:

<nombre-de-dominio>	<TTL>	IN MX	<PRIORITY>	<nombre-de-dominio tipo A>

No pueden apuntar directamente a direcciones IP, sino que tienen que apuntar a entradas de nombre de dominio (A):

<FQDN>			<TTL>	IN A	<Dirección IP>

La prioridad dice en qué orden deberían consultarse, menos prioridad se consulta primero.

El tiempo de vida (TTL) es la cantidad de segundos que pueden esperar los servidores antes de actualizar el nombre de dominio.

Para apuntar al dominio raíz en la zona se puede utilizar @. Por ejemplo, si mi dominio es dominio.com el siguiente registro significa que dominio.com apunta a la dirección 1.2.3.4.

@	10800	IN A	1.2.3.4
