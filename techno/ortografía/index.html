Si administras proyectos usando git y quieres mantener la calidad de tus proyectos puedes mejorar la calidad de entrega usando revisiones automáticas.

Un proceso de calidad:

  • Genera una buena impresión acerca de tu trabajo.

  • Invita a la colaboración.

  • Muestra que eres un profesional.

Los errores sencillos afectan gravemente la credibilidad de un proyecto y por eso estuve buscando herramientas para verificar automáticamente la ortografía en mis proyectos digitales.

Revisar ortografía usando pre-commit ↑

Por el momento uso la siguiente configuración en .pre-commit-config.yaml

- repo: local
  hooks:
  - id: spellcheck
    name: Revisar ortografía
    entry: execlineb -S0 -c 'pipeline { spellcheck -p .spell-check/exceptions -a ${@} } if -x0 { grep -F "&" } false'
    language: system
    files: \.(md|txt)$
    types: [text]

Para resolver este problema, evalué las siguientes herramientas:

hunspell ↑

Es la biblioteca que usa libreoffice, firefox y Adobe.

Es súper flexible, puede usar varios diccionarios simultáneamente:

hunspell -d en_US,es_MX,de_DE archivo-en-varios-idiomas.md

hunspell sólo permite un directorio personal, donde se guardan las palabras que no se reconocen en el idioma, pero yo utilicé un truco para agregar varios diccionarios personales:

hunspell \
	-d en_US,es_MX,de_DE \
	-p <(cat archivo1 archivo2 archivo3) \
	archivo-en-varios-idiomas.md

La desventaja de este método es que no permite guardar palabras nuevas desde la interfaz de revisión ortográfica.

Otra alternativa para agregar múltiples diccionarios categorizados es aprovechar que se pueden agregar directorios con diccionarios con la variable de entorno DICPATH.

Los diccionarios agregados en DICPATH deben ser pares:

diccionario.aff
diccionario.dic

Como no tenía razón para generar un .aff (que contiene la información léxica del lenguaje) vinculé el .aff de un idioma preexistente:

ln -s /usr/share/myspell/es_MX.aff wordlist/nombres.aff
touch wordlist/nombres.dic

Y utilicé este nuevo diccionario así:

hunspell \
	-d en_US,es_MX,de_DE,nombres \
	-p archivo-para-palabras-nuevas \
	archivo-en-varios-idiomas.md
