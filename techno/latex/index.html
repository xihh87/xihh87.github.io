Por qué amo y odio \LaTeX ↑

LaTeX es un lenguaje de programación para hacer documentos. Idealmente permite separar el formato del contenido pero en la práctica me parece que los documentos fuente de LaTeX son horribles por la misma razón que PHP:

  • intenta resolver a la vez función y presentación de la información
  • mezcla niveles de abstracción

Me gusta usar pandoc para separar el contenido completamente al escribir en Markdown, y usar una plantilla con el formato en LaTeX que se encarga del formato.

Compilar al cambiar usando latexmk:

latexmk -pvc -pdf -pdflatex=[programa] archivo.tex

LaTeX surge del trabajo de un par de genios y una gran comunidad ↑

Todo el software tiene errores, con la posible excepción de las cosas escritas por Daniel J. Bernstein y Donald Knuth.

Sabiduría popular

Hace un tiempo Donald Knuth escribió un lenguaje de programación para hacer libros de alta calidad. Una de sus ventajas más grandes es que permite concentrarse en el contenido y automatizar la presentación. El lenguaje (\TeX{}) permite controlar todos los aspectos del documento, pero es bastante complejo.

Leslie Lamport agregó funciones útiles para la edición cotidiana y así hizo más sencillo el lenguaje. Esta variante se llamó \LaTeX{}, es muy popular y produce documentos de alta calidad.

En general produce documentos hermosos ↑

Sin embargo se le nota la edad, ↑

LaTeX nació mucho antes de que se resolvieran los problemas de la internacionalización del texto de manera eficiente y el uso de tipografías fuera generalizado.

Por eso

  • No usa UTF-8.
  • No usa tipografías del sistema.
  • Es terriblemente lento.

Las dos primeras objeciones se pueden remediar usando xelatex y lualatex

tiene una sintaxis horrible, ↑

El código bien escrito puede leerse como prosa.

Hay lenguajes que incitan a la legibilidad pero LaTeX y PHP no pueden hacerlo porque mezclan el contenido y la forma.

  • Es difícil acomodarlo en bloques legibles.
  • Usar espacios en el código a veces evita que el código funcione. (no es independiente de la indentación.)

es extremadamente lento, ↑

Usar un disco de estado sólido o un sistema de archivos en RAM para compilar puede optimizar tu compilación de minutos a segundos. Si usas RAM, recuerda sacar de allí los archivos que quieres conservar.

Usar tipografías del sistema ↑

Para utilizar Unicode con xelatex, se necesita una tipografía que tenga disponibles los glifos para Unicode (p. e. libertine) y agregar al documento:

\usepackage{fontspec}
\usepackage{libertine}

Resolución de errores ↑

Se puede usar un fallback para permitir el uso de caracteres no incluidos en la tipografía elegida:

header-includes: |
  \setmainfont{Latin Modern Sans}
  \usepackage{combofont}
  \usepackage{fontspec}
  \setupcombofont{multiscript-regular}{
    {file:lmsans10-regular.otf:\combodefaultfeat} at #1pt,
    {file:DejaVuSans.ttf} at #1pt,
    {file:NotoSansCJK-Regular.ttc(0)} at #1pt
  }{
    {} ,
    fallback,
    fallback
  }
  \DeclareFontFamily{TU}{multiscript}{}
  \DeclareFontShape {TU}{multiscript}{m}{n}{<->combo*multiscript-regular}{}
  \fontfamily{multiscript}\selectfont

Se pueden usar SVG en LaTeX ↑

El paquete svg usa inkscape para transformar los SVG a PDF o PS.

\documentclass{article}
\usepackage{svg}
\begin{document}
  \includesvg{path/to.svg}
\end{document}
