Agent Guidelines ↑

Build/Lint/Test Commands ↑

  • Build: npm run build or yarn build
  • Lint: npm run lint or yarn lint
  • Test all: npm test or yarn test
  • Test single: npm test -- <test-file> or yarn test <test-file>

Code Style Guidelines ↑

Imports ↑

  • Use ES6 imports with named exports preferred
  • Group imports: external libraries, then internal modules
  • Sort imports alphabetically within groups

Formatting ↑

  • Use consistent indentation (2 spaces or 4 spaces)
  • Max line length: 80-100 characters
  • Trailing commas in multi-line structures

Types ↑

  • Use TypeScript for type safety
  • Prefer interfaces over types for object shapes
  • Use union types for related variants

Naming Conventions ↑

  • Functions: camelCase
  • Classes/Components: PascalCase
  • Constants: UPPER_SNAKE_CASE
  • Files: kebab-case or camelCase

Error Handling ↑

  • Use try/catch for async operations
  • Return Result types or throw descriptive errors
  • Log errors with context information

General ↑

  • Write self-documenting code
  • Use meaningful variable names
  • Keep functions small and focused
