Instalador automático para servidores

La manera más simple de resolver el problema ↑

Posiblemente la forma más simple de hacer un instalador a mi medida sería:

  • ☐ Hacer una imagen mínima personalizada

  • ☐ Hacer un script de configuración automático

    • Generar una partición /EFI.
    • Generar una partición adicional y cifrar el disco.
    • Actualizar base de datos de paquetes.
    • Instalar una base.
    • Ejecutar mis scripts de configuración.

El costo de aprender productos en vez de paradigmas ↑

Cada sistema parece tener su sistema automático, por poner ejemplos:

  • Packer + Terraform + Ansible.
  • Terraform + Ansible.
  • Debian tiene preseed y Fully Automated Install.
  • Ubuntu autoinstall.
  • Suse AutoYast..
  • Red Hat anaconda.
  • Arch tiene arch blocks. (pero puedes hacerte un pequeño script de bash).

Todos estos sistemas tienen su propia configuración y hacen lo mismo con diferentes métodos.

Y tienen que ser bastante complejos para atender las necesidades de sus muchos usuarios.

Cada que quieres usar uno de ellos tienes que invertir varias horas para aprender cómo se hace cada cosa en este sistema particular.

Si aprendes un sistema particular sin entender los principios, estás limitado a ser un técnico y eres dependiente del sistema.

Si conoces los principios sobre los que se fundamentan los sistemas, puedes saber qué buscar y decidir si invertir tiempo en resolver los problemas de esta forma o resolver los problemas de alguna otra.

Conocer los principios también te ayuda a automatizar (o a resolver una vez y auto-documentar).

Construir un sistema nuevo tiene sus costos y cada sistema nuevo va a fallar en los casos particulares para los que no fue diseñado.

Las ventajas contra el costo de automatizar ↑

Automatizar sólo es útil si resuelve un problema valioso, requiere mucho más esfuerzo que hacer las cosas manualmente una única vez porque se tiene que:

  • Hacer en el orden correcto
  • Verificar que funciona en diferentes condiciones
  • Verificar que funciona sin entrada de información

Cuando una persona está realizando una actividad puede reaccionar instantáneamente para resolver los problemas que se generan. Un sistema automático sólo puede reaccionar si esos problemas están previstos.

Auto-documentación como lo mejor de ambos mundos ↑

Si tienes un sistema para registrar todo lo que haces incluyendo las fuentes de información que consultas puedes usar este sistema para resolver los problemas de la forma más chapucera e interactiva, generas un proceso que resuelve el problema.

Es más fácil limpiar este proceso que generarlo interactivo desde cero.

Pensando en esa forma de generar mis scripts de instalación, probablemente podría probar mis scripts en una máquina virtual utilizando el modo de terminal y grabando el resultado.
