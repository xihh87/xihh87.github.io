CRLF en git

En archivos de texto es buena idea utilizar el separador de líneas que usa sistema operativo (en realidad, sería mejor idea homologar el uso del separador de líneas) para que los archivos de texto puedan usarse en todas las aplicaciones.

git tiene una función para convertir los archivos automáticamente entre sistemas:

core.autocrlf

En archivos que sólo son texto es una gran idea pero en otros archivos podría ocasionar corrupción de datos. Por eso git verifica que este proceso automático no vaya a cambiar los datos en el directorio de trabajo.

Si al recuperar los datos desde el repositorio cambiaría lo que hay en el directorio de trabajo, git se pone triste:

fatal: CRLF would be replaced by LF in archivo-problemático

Hay varias posibles soluciones al problema:

  • Desactivar la transformación automática: git config core.autocrlf false.

  • Transformar los datos en el directorio de trabajo: dos2unix archivo-problemático.

  • Ignorar que se podrían romper los datos, pero pedirle a git que nos avise: git config core.safecrlf warn

  • Ignorar que se podrían romper los datos: git config core.safecrlf false
