Medir tiempo de trabajo en proyectos con timew

  • Anotar cosas en la bitácora de trabajo
  • Otras alternativas

Yo uso timewarrior, paquete timew en Arch, BTW.

  1. Al empezar a trabajar en una tarea:

    timew track \
    "Descripción de la actividad" \
    +proyecto

  2. Al terminar la tarea o dejar de trabajar en ella.

    timew stop

  3. Si continúo sobre la última tarea que se detuvo.

    timew continue

    También puede continuarse una tarea anterior:

    timew summary \
    from yesterday to now :ids
timew continue @${TASK_ID}

  4. Al terminar el proyecto puedo reportar el tiempo copiando la salida de:

    timew summary \
    from ${PROJECT_START} to ${PROJECT_END}

Anotar cosas en la bitácora de trabajo ↑

El manual de timewarrior especifica dos instrucciones para asociar a las tareas: timewarrior annotate y timewarrior tag.

No son equivalentes pero no está documentado cómo usarlo o en qué difieren.

Cada tarea que se guarda en un archivo de texto con el siguiente formato:

inc <tiempo_inicio> - <tiempo_fin> # "etiqueta", "etiqueta larga" # "anotación" "anotación larga"

Al usar comandos de visualización como task summary:

timew summary

Wk  Date       Day Tags                                        Start      End    Time   Total
--- ---------- --- ---------------------------------------- -------- -------- ------- -------
W13 2022-03-31 Thu etiqueta, etiqueta larga                 10:00:00 10:15:00 0:15:00
                   etiqueta de la siguiente tarea.          10:17:14 13:49:04 3:31:50 3:46:50

                                                                                      3:46:50

Por lo que puedo observar, las etiquetas se contabilizan y pueden buscarse, pero no sé cómo recuperar o usar las anotaciones, si es posible.
