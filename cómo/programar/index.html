Cómo programar

“El trabajo de un ingeniero de software no es escribir código, es resolver problemas.”

  1. Definir el problema.

  2. Buscar si alguien más ya resolvió el problema.

  3. Escribir en lenguaje humano cómo resolveríamos el problema.
    Esta es la forma más barata de diseño, no es tan pesado como la fase de diseño Waterfall, pero permite pensar cómo resolver el problema, diseñar interfaces, identificar incógnitas.

  4. Describir mejor hasta que se pueda traducir fácilmente a lenguaje de máquina.

    De esa descripción nacerán los módulos que se convertirán en funciones, y habrá nombres para las variables, clases y otros elementos.

  5. Traducirlo para la computadora.

  6. Borrar los comentarios que describen el cómo y no el porqué.

    “El lugar para describir cómo se hacen las cosas es el código.”

  7. Repetir desde el paso 4.
