Principios FAIR ↑

Son guías, no

Findable:

  • Localizador único
  • Metadatos enriquecidos
  • Repositorios indizados
  • Identificador persistente en los metadatos

Accesible.

  • Normas internacionales e interoperables
  • Formatos no propietarios
  • Autenticación para datos con acceso restringido
  • Metadatos siempre disponibles

Interoperable.

  • Vocabularios reconocidos
  • Vocabularios disponibles
  • Metadatos relacionados (descripción y preservación) Incluye los datos complementarios.

Reusable.

  • Varios atributos
    • protocolos experimentales
    • fabricante
    • marca
    • especies usadas
    • régimen de drogas
  • Licencias de uso
  • Procedencia
    • origen
    • a quién citar y cómo
    • cómo fueron procesados
    • fueron publicados antes
    • se usan datos de terceros
  • Normas de comunidad
    • vocabulario común
    • almacenamiento e intercambio

Hay que considerar el impacto a largo plazo de los datos.

Las bibliotecas son un lugar de apoyo profesional.

Muchos investigadores aprenden a gestionar sus datos en el trabajo.

Los bibliotecarios pueden validar y gestionar los datos.

  1. Identificar los datos dentro del ámbito de la investigación.
  2. Obtener los datos.
  3. Seleccionar, limpiar, corregir, normalizar y preparar los datos.
  4. Hacer que los datos sean localizables y utilizables.
  5. Asegurar que estarán disponibles por un tiempo útil.

Los datos son:

  • recopilados durante una investigación
  • analizados, manipulados, procesados y visualizados -> Estructura
  • sustentan resultados de una investigación
  • aceptados como válidos por una comunidad de investigación
  • reusables -> Licencias abiertas

URI permanente, como doi

  1. Accesible -> licencias abiertas
  2. Estructurado
  3. No propietario -> CSV, JSON, XML
  4. URL persistentes
  5. Red -> Enlazado a otros datos para dar contexto

Gestión de los datos y sus versiones ↑

jerárquico:

Familiar y utilizado Bueno para representar la estructura Archivos similares se almacenan juntos Subcarpetas fungen como listas relacionadas

  • difícil equilibrio entre amplitud y profundidad
  • duro de establecer y mantener
  • archivos en un único lugar
  • difícil de reorganizar

etiquetas:

Archivos en más de una categoría Múltiples tipos de categorías Más fácil y rápido que estructura jerárquica Más fácil de combinar que sistemas jerárquicos

incompatible con sistemas operativos riesgo de etiquetado inconsistente difícil encontrar material mal etiquetado menos conveniente para la estructura de información

Propuesta para nombrar archivos:

número / nombre / acrónimo -> proyecto nombre / siglas -> investigador versión fecha o rangos de fechas coordenadas geoespaciales

Evitar valores que no sean alfanuméricos y/o -, _, ..

Usar ceros a la izquierda para la numeración secuencial.
