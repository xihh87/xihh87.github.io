Requerimientos ↑

La petición es “Herramienta de web-scrapping para revisar en tiempo real la popularidad de los proyectos de la semana de ingenierías”. Esto tendría que estar listo para 2024-05-02 y hoy es 2024-05-25, por lo que hay 7 días exactamente para hacerlo.

Evaluación de factibilidad ↑

web-scrapping va en contra de los términos de uso de las RRSS, por lo que se podrían bloquear las sesiones, aplicaciones o IPs que utilicen este método.

Se podría hacer usando las APIs de cada gestor de redes sociales (Fb, IG, twt).

La funcionalidad tendría que hacerse mediante polling, o notificationes (más eficiente, más difícil) (ejemplo 1, ejemplo 2).

Las llamadas podrían hacerse desde el cliente web (pero eso haría imposible proteger las API keys) o desde un back-end que permita entregar esta información a un cliente web (preferible).

Requisito previo a la implementación ↑

  • Registrar tokens API en cada servicio:
    • Facebook
    • Instagram
    • Twitter
    • (Opcionalmente) otros
  • Registrar los posts donde se está promocionando cada equipo.

Propuesta segura ↑

La forma más sencilla de implementar tal servicio de forma segura es:

  • Hacer un sistema de scheduling que los revise por medio de la API cada cierto tiempo.
    • (Alternativamente) Configurar los sistemas de notificación hacia esos posts.
  • Hacer un sistema en el cliente web que solicite la información al servicio de consulta cada cierto tiempo.
    • (Alternativamente) Generar un sistema de notificación que envíe la información a los clientes conectados.

Propuesta insegura ↑

  • Generar código que cliente que haga la solicitud API cada cierto tiempo para los sistemas indicados en cada página.

La desventaja de esta propuesta es que cualquier persona con los suficientes conocimientos técnicos podría usar nuestra API para hacer consultas, con el subsecuente costo el uso de la API e interrupcción del servicio si hay demasiadas peticiones (legítimas y/o ilegítimas).
